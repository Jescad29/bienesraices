        Modelo en MVC: Encargado de todas la interacciones en la base de datos, obtener datos, actualizarlos y eliminar.
            - El modelo se encarga de consultar una base de datos, obtiene la informaciÃ³n,

            - El modelo tampoco se encarga de actualizar la informaciÃ³n directamente; es el 
              Controlador quien decide cuÃ¡ndo llamarlo.
        Vista en MVC: Se encarga de todo lo que se ven en pantalla (HTML).
            - Node soporta mÃºltiples Template Engine como son EJS, Pug, Handlebars.
            - Pero si utilizamos Node como API e implementamos React, Vue, Angular y Svelte, estos serÃ­an tu vista.
            - El Modelo consulta la base de datos, pero es por medio del Controlador que se decide que Vista hay que llamar y que datos presentar.
        Controlador en MVC: Es el que comunica modelo y vista; antes de que el Modelo consulte, la base de datos el Controlador es el encargado de llamar un Modelo en especifico. 
            - Una vez consultado el Modelo, el controlador recibe esa informaciÃ³n, manda llamar a la vista y le pasa la informaciÃ³n.
            - El controlador es el que manda a llamar la vista y modelos, que se requieren en cada parte de tu aplicaciÃ³n.

        soportar nuestra aplicaciÃ³n.
